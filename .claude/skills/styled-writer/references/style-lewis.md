# Michael Lewis 风格规范

## 选题聚焦

金融、体育、科技、官僚系统中"少数看懂规则的人"，以及制度漏洞、群体盲点和系统失灵。核心问题是：为什么多数人视而不见，少数人却提前看清？重点不是事件本身，而是规则的真实运作方式与人们选择性的盲目。

## 分析框架

「人物驱动 + 系统解剖」。通过具体人物的经历、性格和决策过程，揭示一个行业或制度如何运作——尤其是那些内部人觉得理所当然、外部人觉得荒谬的规则。强调以下维度：
1. **信息不对称**：少数人掌握了什么多数人不知道的？
2. **激励机制**：系统中的人被什么利益/恐惧驱动？
3. **组织文化**：共识如何形成，异见如何被排除？
4. **人性弱点**：贪婪、自负、群体压力与自我欺骗

核心不是做道德审判，而是展示「人们如何在错误的系统中做出自认为合理的选择」。

## 叙事方式

人物先行。找到系统中那个「格格不入的人」——太执拗、太聪明、太诚实或太偏执——让这个人的视角成为读者的入口。结构上频繁在「人物故事—专业机制—社会背景」之间切换，通过对话、细节和冲突推进。像侦探小说一样揭开层层内幕。

## 语言风格

机智、锐利、节奏快。善用黑色幽默、讽刺和精准比喻（如把次贷比作「金融界的疯牛病」），把复杂机制讲得极易理解。既有《华尔街日报》式的调查感，也有小说式的人物塑造。语气带一点「你不会相信这事竟然这样运作」的惊讶感，但保持冷静叙述。

## 文章结构

```
开头：一个戏剧化场景或一个「怪人」的出场 → 制造好奇
展开：逐步揭示行业规则与隐藏逻辑（为什么这个系统如此奇怪？）
中段：人物故事 × 机制解剖 × 社会背景交替推进
高潮：冲突爆发或真相暴露的时刻
结尾：回到人物命运，指向更广泛的制度问题或时代寓言
```

禁止使用小标题。用人物视角的自然切换推进叙事。

## 输出要求

- 必须有鲜明人物、真实场景和制度细节
- 必须让读者理解一个复杂系统的真实运作逻辑
- 必须包含认知反转或「内幕揭示」感——读者读完觉得「原来是这样运作的」
- 避免空泛商业术语、纯数据堆砌、无人物的宏观分析、道德说教，以及只有观点没有故事
