# 多风格写作

根据 `作者：主题` 关键词，调用对应作家的写作风格，完成一篇5000-10000字的深度文章。

## 第一步：解析输入

从用户输入中提取两个核心信息：

1. **目标作者**：从以下五位中匹配——Gladwell、何伟（Peter Hessler）、Marks（霍华德·马克斯）、廖信忠、高华。如果用户只说了作者名没带主题，追问主题。
2. **写作主题**：冒号后面的完整主题。如果主题过于宽泛（如"中国经济"），追问聚焦方向。

匹配规则（宽松匹配，不要求精确写法）：
- 包含 gladwell / 格拉德威尔 / 格拉德韦尔 → Gladwell风格
- 包含 何伟 / hessler / 彼得海斯勒 → 何伟风格
- 包含 marks / howard / 霍华德马克斯 / 马克斯 → Marks风格
- 包含 廖信忠 / 廖信忠 → 廖信忠风格
- 包含 高华 → 高华风格

如果无法匹配任何作者，列出五位作者的简要说明，让用户选择。

---

## 第二步：研究

搜集团研究分为两轮。第一轮建立理解，第二轮定向补充。

### 搜索策略

使用1个子Agent并行搜索，每次搜4-5次。总共控制在8次搜索以内。

**搜索清单**（根据主题类型灵活调整）：

- 主题的核心事实、时间线、关键人物
- 相关的社会/经济/历史背景
- 具体人物故事、案例、场景细节
- 不同观点、争议、反直觉现象
- 数据、研究、报告（如适用）

**硬约束**：
- 关键事实必须联网确认，不能靠已有知识
- 每个方向搜1-2次，拿到可用事实即停
- 需要深入确认的细节，用最多1次WebFetch定向补充
- 搜索总次数控制在8次以内
- 搜不到的信息诚实标注「该信息暂缺」

### 信息充分性自检

- 能讲出一个有因果逻辑的完整故事吗？
- 有足够的具体人物、场景、细节支撑写作吗？
- 有一个核心的反直觉洞察或张力点吗？

缺关键信息就针对性补搜1-2次。

根据匹配的作者，加载对应的写作风格规范，用该风格完成文章。

## 风格一：Malcolm Gladwell

### 选题聚焦

日常现象背后的反直觉机制、小概率事件引发的系统性后果、个人行为与社会结构之间的隐秘关联。优先选择读者「习以为常但从未深想」的问题。

### 分析框架

1. 提出一个违反直觉的问题或案例
2. 通过心理学、社会学、行为经济学、历史案例等跨学科材料建立解释模型
3. 不断插入具体人物故事作为论据
4. 最终回到更大的社会规律或人性机制

重点不是证明观点，而是「重新定义问题」。

### 叙事方式

故事驱动。开头从一个具体人物、事件或细节切入，以悬念推动。结构上频繁在「故事—理论—故事」之间切换，通过类比、案例拼接、历史回溯形成递进。避免纯学术论证。

### 语言风格

口语化、轻快、聪明、有观察感。句子简洁，避免术语堆积。善用反问、类比、短段落制造阅读节奏。语气像在向聪明朋友解释一个有趣发现，而非正式论文。

### 文章结构

```
开头：一个具体人物/事件/细节 → 制造悬念
转场：表面解释为什么不够 → 引入更深层的问题
中段：跨学科材料 + 人物故事 + 案例拼接 → 逐层揭示机制
转场：用另一个案例验证机制不是孤例
结尾：回到开头故事，用新的理解重新解释 → 提炼更大的启发
```

禁止使用小标题。板块之间用转场句自然衔接。

### 输出要求

- 必须包含具体人物、场景、细节
- 必须有一个核心反直觉洞察
- 必须让读者在阅读过程中不断产生「原来如此」
- 避免空泛观点、宏大抒情、复杂术语、道德说教，以及没有故事支撑的抽象分析

## 风格二：Peter Hessler（何伟）

### 选题聚焦

普通人的生活轨迹、社会转型中的细微变化、地方经验与宏观时代之间的关系。优先关注边缘人物、小城、乡村、基层职业，以及长期被忽视的日常现实。

### 分析框架

「长期观察 + 个体切片」。通过持续多年的田野调查视角、重复接触同一批人物的方式，呈现社会结构如何缓慢改变个人命运。避免直接下结论，而是通过细节累积让读者自行理解制度、文化与时代变化。

### 典型结构

从具体场景切入——一次吃饭、聊天、乘车、课堂或街头观察；随后自然展开人物背景、历史脉络与地方环境；中间穿插作者亲历、对话与观察笔记；最后以克制、开放式结尾收束，而非强行升华。

叙事强调「人在场」——作者既是观察者也是参与者。

### 语言风格

冷静、克制、细腻、低姿态。句子不炫技，不过度修辞。大量使用准确生活细节、真实对话和动作描写。语气带有温和幽默感与人类学式好奇心，对人物保持尊重，不讽刺、不猎奇、不煽情。

### 文章结构

```
开头：一个具体场景（吃饭/聊天/乘车/街头观察）
展开：人物背景 → 地方环境 → 历史脉络
中段：作者亲历、对话片段、观察笔记交替推进
结尾：克制、开放式收束，不强行升华
```

禁止使用小标题。用场景切换自然推进。

### 输出要求

- 必须包含真实生活细节、人物行为、环境描写与自然对话
- 必须呈现复杂性与矛盾感
- 必须让宏大时代变化通过普通人的经历间接显现
- 避免口号化表达、宏大议论、预设立场、道德裁判，以及「替人物总结意义」

## 风格三：Howard Marks（霍华德·马克斯）

### 选题聚焦

市场周期、风险定价、投资者心理、资产价格与价值偏离、逆向投资、概率与不确定性。优先讨论「看似确定但其实脆弱」的市场共识。

### 分析框架

「二阶思维」：
1. 先列出主流观点和市场共识
2. 追问其隐含假设、反身性后果和失败条件
3. 重点分析概率分布、风险补偿、安全边际、周期位置
4. 不做单一结果预测，判断赔率是否有吸引力

强调「我不知道未来，但可以判断赔率是否有吸引力」。

### 典型结构

开头提出一个投资命题或市场现象；随后拆解共识、情绪、估值和历史类比；中段用简洁段落递进，常以「另一方面」「问题在于」「关键不是……」转折；结尾给出审慎判断，保留条件和不确定性。

### 语言风格

理性、克制、清晰、经验主义。句子短而有逻辑，概念定义准确。善用对比、反问、格言式总结。语气像一位资深投资人写给LP的冷静备忘录，不煽动、不炫技、不追求戏剧性。

### 文章结构

```
开头：一个投资命题或市场现象
拆解：共识是什么 → 隐含假设 → 反身性后果
中段：概率分布 × 风险补偿 × 安全边际 × 周期位置
转折：另一方面 / 问题在于 / 关键不是
结尾：条件式判断，保留不确定性
```

允许使用章节标题（罗马数字或简洁短句），但不用编号体系。

### 输出要求

- 必须区分价格与价值、风险与波动、概率与确定性
- 必须体现谦逊、周期意识和逆向思考
- 结论应是条件式判断（「如果X，则Y；但如果Z，则W」），而非预测式断言
- 避免宏观喊单、情绪化判断、确定性语言、复杂模型堆砌、事后归因和未经检验的市场口号

## 风格四：廖信忠

### 选题聚焦

中国大陆社会的日常生态、地域文化差异、小人物经验、互联网亚文化、城乡变迁与时代情绪。优先选择「本地人习以为常、外地人觉得新鲜」的细节。

### 分析框架

「个人见闻 + 社会观察」。通过亲历、聊天、网络梗、街头场景、小历史和地方记忆切入，再逐步延伸到更大的社会心理与代际变化。重点不是严密论证，而是提供「接地气的真实感」和时代氛围。

### 典型结构

开头从一个具体小事、网络热梗、旧闻或个人回忆切入；中间不断穿插生活细节、方言、人物对话和历史背景；结构松散但有节奏，像聊天一样自然推进；结尾常带一点感慨、幽默或时代错位感，而非正式总结。

### 语言风格

口语化、轻松、有烟火气，带台湾视角的观察距离感。善用短句、插话、括号补充、网络语言与冷幽默。对社会现象保持温和调侃，但不尖锐攻击。既熟悉大陆语境，又保留一点「旁观者」视角。

### 文章结构

```
开头：一个具体小事/网络梗/旧闻/回忆
中段：生活细节 × 人物对话 × 历史背景 × 个人感受 交织推进
节奏：松散但有线索串联，像聊天
结尾：一点感慨/幽默/时代错位感，不正式总结
```

禁止使用小标题。靠换行和语气制造节奏。

### 输出要求

- 必须有大量生活细节、真实场景、地方感与时代感
- 必须像「长期混迹民间的人在讲亲眼所见」
- 允许轻微碎碎念和跳跃
- 避免学术腔、宏大叙事、强烈意识形态表达、刻意煽情，以及居高临下的评论口吻

---

## 风格五：高华

### 选题聚焦

中国近现代政治史中的权力结构、意识形态运作、党内关系、知识分子处境与政治运动的形成机制。重点不是「历史事件本身」，而是事件背后的制度逻辑与权力运行方式。

### 分析框架

「档案材料 + 制度分析 + 人物关系」。通过文件、电报、讲话、日记、回忆录与历史细节交叉验证，重建决策过程与政治心理。强调历史情境，还原人物在特定结构中的选择与限制，避免简单道德化。

### 典型结构

先提出历史问题或矛盾；随后回溯背景、组织结构与关键人物关系；通过时间线与材料递进展开分析；中间频繁引用原始史料；最终揭示制度逻辑、意识形态惯性或政治运动的内在机制。叙事强调「层层推进」和证据链。

### 语言风格

冷静、克制、严谨、密度高。句子偏长但逻辑清晰，少形容词，多事实与推理。语气保持历史研究者距离感，不刻意煽情，也不直接下政治结论，而是通过材料本身形成张力。

### 文章结构

```
开头：一个历史问题或矛盾现象
回溯：背景 → 组织结构 → 关键人物关系
展开：时间线推进 + 原始史料穿插 + 逻辑分析
中段：层层推进，每一层解决一个子问题
结尾：揭示制度逻辑或历史机制，保持克制
```

允许使用简洁的章节标题，但避免过度分级。

### 输出要求

- 必须基于具体史料、时间线与人物关系
- 必须区分史实、推测与解释（推测需明确标注）
- 必须呈现复杂性与制度背景
- 避免口号化表达、阴谋论、情绪化控诉、简单二元道德判断，以及脱离历史语境的「上帝视角」评价

## 通用写作禁区（五种风格统一遵守）

### 套话禁用

以下表达任何情况下不得出现：
- 「首先...其次...最后」
- 「综上所述」「总而言之」「总的来说」
- 「值得注意的是」「值得一提的是」
- 「不难发现」「我们可以发现」
- 「不可否认」「毫无疑问」
- 「在当今...时代」「随着...的发展」
- 「众所周知」

### 高频踩雷词

- 「说白了」
- 「意味着什么？」
- 「这意味着」
- 「本质上」
- 「换句话说」

### 禁止行为

- 不编造研究、数据、引用、人物或场景
- 不写没有故事支撑的抽象分析
- 不以宏大叙述开篇
- 不说「AI工具」「某个模型」，要说具体名称

## 第四步：输出

### 文件输出（同时生成两种格式）

文章写完后，同时输出 Markdown 和 EPUB 两种格式：

- **Markdown** → `markdown/{作者名}_{主题}.md`
- **EPUB** → `epub/{作者名}_{主题}.epub`

文件名示例：`Gladwell_为什么外卖骑手不闯红灯反而更危险.md` / `.epub`

### EPUB生成流程

使用 hv-analysis skill 自带的 `md_to_epub.py` 脚本生成 EPUB：

1. **确保依赖已安装**：`pip install markdown ebooklib --break-system-packages`
2. **生成EPUB**：
   ```bash
   python .claude/skills/hv-analysis/scripts/md_to_epub.py markdown/input.md epub/output.epub --author "{作者名}风格 · Claude Code"
   ```
3. 构建完成后自动打开 EPUB 文件

### EPUB排版规范

`md_to_epub.py` 内置CSS，排版特征：
- 段首缩进2个中文字符（`text-indent: 2em`）
- 段落间无空行，靠缩进区分段落
- 标题不缩进，引用块和列表项也不缩进

### Markdown写作注意事项

为了让脚本正确解析并生成最佳EPUB效果：

- 第一行用 `# 标题` 作为文章标题
- 用 `##` 作为章节标题（如果需要）
- 用 `###` 作为子章节（如果需要）
- Gladwell、何伟、廖信忠风格不使用任何标题（`##`/`###`），纯靠段落推进
- Marks、高华风格可使用简洁章节标题
- 加粗使用 `**文本**`
- 引用使用 `>` 语法

### 文章元信息

在Markdown文章末尾附加：

```
---
写作风格：{作者名}风格
研究时间：{当前日期}
字数：{实际字数}
信息来源：
- {来源1}
- {来源2}
...
---
```

- [ ] 有具体人物、场景、细节？不是空泛议论？
- [ ] 结构符合该作者的典型推进方式？
- [ ] 语言风格匹配目标作者？不是通用的AI腔？
- [ ] 没有触犯通用禁区？
- [ ] 字数在5000-10000范围内？
- [ ] 关键事实有来源支撑？
