# 多风格写作

根据 `作者：主题` 关键词，调用对应作家的写作风格，完成一篇5000-10000字的深度文章。

## 第一步：解析输入

从用户输入中提取两个核心信息：

1. **目标作者**：从以下八位中匹配——Gladwell、何伟（Peter Hessler）、Marks（霍华德·马克斯）、廖信忠、高华、Lewis（Michael Lewis）、扶霞（Fuchsia Dunlop）、大象公会。
2. **写作主题**：冒号后面的完整主题。如果主题过于宽泛（如"中国经济"），追问聚焦方向。

### 匹配规则（宽松匹配，不要求精确写法）

- 包含 gladwell / 格拉德威尔 / 格拉德韦尔 → Gladwell风格
- 包含 何伟 / hessler / 彼得海斯勒 → 何伟风格
- 包含 marks / howard / 霍华德马克斯 / 马克斯 → Marks风格
- 包含 廖信忠 / 廖信忠 → 廖信忠风格
- 包含 高华 → 高华风格
- 包含 lewis / michael / 迈克尔刘易斯 → Lewis风格
- 包含 扶霞 / dunlop / 邓洛普 / 鱼翅与花椒 → 扶霞风格
- 包含 大象公会 / 大象 → 大象公会风格

### 未指定作者时：自动路由

如果用户输入中没有明确指定作者（只有主题，没有"作者："前缀），按以下逻辑分析主题，推荐2-3位最合适的作者供用户选择：

**路由判断逻辑**（按主题特征匹配）：

| 主题特征 | 推荐作者 | 适配理由 |
|---------|---------|---------|
| 金融/投资/市场/交易/估值/周期 | Marks 或 Lewis | 偏投资分析与理念 → Marks；偏制度失灵/人物故事/内幕 → Lewis |
| 制度漏洞/官僚系统/组织失灵/告密者/怪人 | Lewis | 人物驱动系统解剖，最佳适配 |
| 体育/竞技/选秀/交易策略 | Lewis | Lewis的经典领域 |
| 科技公司/创业/硅谷/产品决策 | Lewis 或 Gladwell | 偏人物与系统 → Lewis；偏行为机制 → Gladwell |
| 社会现象/行为心理/认知偏差 | Gladwell | 反直觉机制解释 |
| 中国小城/基层人物/城乡变迁/教育 | 何伟 | 长期观察+个体切片 |
| 中国政治史/制度运作/权力结构 | 高华 | 档案分析+制度逻辑 |
| 市井文化/网络亚文化/地域差异/消费潮流 | 廖信忠 | 烟火气+个人见闻 |
| 饮食/食物/厨师/地方菜系/食材 | 扶霞 | 饮食文化+社会观察 |
| 冷知识/流行误解/历史考据/反常识 | 大象公会 | 知识拆解+资料考据 |

**路由流程**：
1. 分析主题关键词，判断属于哪个/哪些领域
2. 按上表选出2-3位最适配的作者
3. 用一句话简述每位作者的适配理由
4. 列出选项让用户选择，格式示例：

```
这个主题可以从几个角度切入：
- **Lewis风格**：如果有某个"看清规则的人"作为主角，可以写成一个系统解剖故事
- **Gladwell风格**：如果重点是这个现象背后的反直觉机制和行为心理
- **大象公会风格**：如果更偏知识考据，拆解流行误解

你倾向哪个？
```

如果主题跨领域特征明显（如"中国餐饮行业的资本化"），同时推荐不同角度的作者（扶霞写饮食文化 × Lewis写资本运作），让用户决定叙事角度。

## 第二步：研究

搜集团研究分为两轮。第一轮建立理解，第二轮定向补充。

### 搜索策略

使用1个子Agent并行搜索，每次搜4-5次。总共控制在8次搜索以内。

**搜索清单**（根据主题类型灵活调整）：

- 主题的核心事实、时间线、关键人物
- 相关的社会/经济/历史背景
- 具体人物故事、案例、场景细节
- 不同观点、争议、反直觉现象
- 数据、研究、报告（如适用）

**硬约束**：
- 关键事实必须联网确认，不能靠已有知识
- 每个方向搜1-2次，拿到可用事实即停
- 需要深入确认的细节，用最多1次WebFetch定向补充
- 搜索总次数控制在8次以内
- 搜不到的信息诚实标注「该信息暂缺」

### 信息充分性自检

- 能讲出一个有因果逻辑的完整故事吗？
- 有足够的具体人物、场景、细节支撑写作吗？
- 有一个核心的反直觉洞察或张力点吗？

缺关键信息就针对性补搜1-2次。

## 第三步：写作

根据匹配的作者，使用已加载的风格规范完成文章。

---

## 通用写作禁区（八种风格统一遵守）

### 套话禁用

以下表达任何情况下不得出现：
- 「首先...其次...最后」
- 「综上所述」「总而言之」「总的来说」
- 「值得注意的是」「值得一提的是」
- 「不难发现」「我们可以发现」
- 「不可否认」「毫无疑问」
- 「在当今...时代」「随着...的发展」
- 「众所周知」

### 高频踩雷词

- 「说白了」
- 「意味着什么？」
- 「这意味着」
- 「本质上」
- 「换句话说」

### 禁止行为

- 不编造研究、数据、引用、人物或场景
- 不写没有故事支撑的抽象分析
- 不以宏大叙述开篇
- 不说「AI工具」「某个模型」，要说具体名称

## 第四步：输出

### 文件输出（同时生成两种格式）

文章写完后，同时输出 Markdown 和 EPUB 两种格式：

- **Markdown** → `markdown/{文章标题}.md`
- **EPUB** → `epub/{文章标题}.epub`

文章标题即 Markdown 中 `# 标题` 的内容。文件名示例：`离广岛三十分钟，有座漂在海上的岛.md` / `.epub`

### EPUB生成流程

使用 hv-analysis skill 自带的 `md_to_epub.py` 脚本生成 EPUB：

1. **确保依赖已安装**：`pip install markdown ebooklib --break-system-packages`
2. **生成EPUB**：
   ```bash
   python .claude/skills/hv-analysis/scripts/md_to_epub.py markdown/input.md epub/output.epub --author "{作者名}风格 · Claude Code"
   ```
3. 构建完成后自动打开 EPUB 文件

### EPUB排版规范

`md_to_epub.py` 内置CSS，排版特征：
- 段首缩进2个中文字符（`text-indent: 2em`）
- 段落间无空行，靠缩进区分段落
- 标题不缩进，引用块和列表项也不缩进

### Markdown写作注意事项

为了让脚本正确解析并生成最佳EPUB效果：

- 第一行用 `# 标题` 作为文章标题
- 用 `##` 作为章节标题（如果需要）
- 用 `###` 作为子章节（如果需要）
- Gladwell、何伟、廖信忠、Lewis、扶霞风格不使用任何标题（`##`/`###`），纯靠段落推进
- Marks、高华、大象公会风格可使用简洁章节标题
- 加粗使用 `**文本**`
- 引用使用 `>` 语法

### 文章元信息

在Markdown文章末尾附加：

```
---
写作风格：{作者名}风格
研究时间：{当前日期}
字数：{实际字数}
信息来源：
- {来源1}
- {来源2}
...
---
```

### 质检清单

交付前自检：
- [ ] 核心理念符合目标作者的风格？不是混搭？
- [ ] 有具体人物、场景、细节？不是空泛议论？
- [ ] 结构符合该作者的典型推进方式？
- [ ] 语言风格匹配目标作者？不是通用的AI腔？
- [ ] 没有触犯通用禁区？
- [ ] 字数在5000-10000范围内？
- [ ] 关键事实有来源支撑？
- [ ] 搜不到的信息诚实标注了「暂缺」？
- [ ] Markdown 和 EPUB 两种格式均已生成？
- [ ] EPUB 已自动打开确认排版无误？
